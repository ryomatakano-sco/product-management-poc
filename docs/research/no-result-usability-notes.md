# 「検索結果なし」パス ユーザビリティテスト台本（no-result usability notes）

**Status:** 計画のみ
**対象:** クリニック受付・物販担当スタッフ 3〜4 名（30 分/名）
**目的:** ルックアップが空振りした瞬間（最頻の現実的モーメント、現状ほぼ
無設計）にユーザーが何をするか・どこで詰まるかを観察する。

## なぜこのテストか

再現率がいくつであれ、**「何も返ってこない」体験は毎日必ず起きる**。
現状の UI は空振り時に薄いメッセージを出すのみで、次の一手
（手入力に切り替える / 名前で再検索 / スキップして後で埋める）への
導線が計られていない。ここが登録時短価値の実質的な下限を決める。

## セッション設計（30 分）

1. **導入（3 分）** — 録画同意。普段の商品登録のやり方を 2 問だけ聞く
   （紙? Excel? 既存レセコン?）。
2. **タスク 1: ヒットする JAN（5 分）** — ウォームアップ。市販歯ブラシの
   JAN でスキャン→自動入力→保存まで。成功体験の基準線を作る。
3. **タスク 2: ヒットしない JAN（10 分・本命）** — 事前に MISS になると
   確認済みの B2B 消耗品（例: グローブの箱）を渡す。
   - 観察: 空振り表示に気づくか / 次に何をクリックするか /
     手入力へ切り替えるまでの秒数 / 諦めの兆候（ため息・放置・助け要求）。
   - **介入しない。** 3 分詰まったらプローブ: 「今なにが起きたと思いますか?」
4. **タスク 3: 名前ルックアップへのリカバリ（7 分）** — JAN で空振り後、
   商品名での再検索に自力でたどり着けるか。
5. **事後インタビュー（5 分）**
   - 「空振りのとき、システムに何をしてほしかったですか?」
   - 「この画面のまま 1 日 20 商品を登録できそうですか?」
   - SEQ（Single Ease Question）: タスク 2 の難しさを 1〜7 で。

## 記録するメトリクス

| 指標 | 取り方 |
|---|---|
| 空振り→手入力切替の所要秒数 | 録画から計測 |
| 自力リカバリ率（タスク 3） | できた/できない |
| SEQ スコア | 各タスク後に口頭で |
| 発話プロトコルの引用 | 空振り瞬間の生発話を最低 1 つ/名 |

## 実施メモ

- 実機（店頭想定なら実際のカウンター高さ・スキャナ）で行う。デモ DB は
  `product_management_dev` のデモデータを流用し、テスト用 JAN を仕込む。
- 4 名で十分（Nielsen の 5 名則。頻出問題の発見が目的で統計ではない）。
- 結果は `docs/research/findings-no-result-<date>.md` に、
  「観察 → 解釈 → UI 変更案」の 3 列で 1 ページに収める。
